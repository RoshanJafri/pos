1. Make sure your repo is initialized and main exists

git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin <your-private-repo-url>
git push -u origin main

2. Create a separate branch for testing

git checkout -b dev

Now you’re working in dev. Any changes here won’t affect main until you merge.
3. Work in dev branch

# make changes
git add .
git commit -m "Test feature"

4. Merge into main when stable

Switch to main, then merge:

git checkout main
git merge dev

Resolve conflicts if any, then:

git push origin main

5. Keep dev updated with the latest main

After deploying or changes in main:

git checkout dev
git merge main
